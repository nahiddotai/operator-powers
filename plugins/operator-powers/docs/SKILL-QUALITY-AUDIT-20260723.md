# Operator Powers Skill Quality Audit

Date: 2026-07-23
Scope: the 23 current skills not covered by the approved review of Customer Insight Synthesizer, Content Performance, Case Study Builder, Content Repurposing, and HTML Slideshow.
Audience lens: non-technical operators, creators, freelancers, small businesses, and ambitious professionals who need useful work, clearer decisions, saved time, trust, or income rather than impressive-looking AI output.

This is an opportunity audit, not a change log. None of the 23 skills below were edited during this pass.

## Overall Finding

The collection is stronger on job boundaries and factual safety than most public skill packs. The recurring quality risk is one level later: a number of skills specify the sections to return but do not make enough of the difficult creative or operational decisions that separate a polished artifact from a competent template.

The most efficient improvement is not to add a large audit rubric to every skill. It is to strengthen three things only where the job needs them:

1. An evidence or input sufficiency decision before drafting.
2. A small craft bar that names what makes this particular output non-generic.
3. One complete, realistic reference example or fixture used in clean-session testing.

## Highest-Priority Opportunities Before Public Release

1. **Instagram Carousel Maker:** replace the fixed template-first approach with source-led narrative and visual direction. This is the clearest current route to generic showcase work.
2. **LLM Council:** reduce the very long, expensive fixed process and make independence claims more precise. The current version can create theatre around a decision rather than better evidence.
3. **De-Slop:** separate universal clarity problems from Nahid-specific style preferences so the skill preserves each customer's voice.
4. **Start Here:** make the privacy explanation exact from its first sentence. The current local-only claim is broader than the later telemetry qualification.
5. **Brand System Builder:** require usable visual tokens and application artifacts, not only a written brand specification.
6. **Digital Lead Magnet Maker:** add a usefulness gate and a finished production standard so the resource earns an email address rather than merely looking complete.

## Skill-by-Skill Opportunities

### [AGENTS.md Setup](../skills/agents-md-setup/SKILL.md)

**Current strength:** practical interview, clear safety boundaries, and a useful 60-line constraint.
**Opportunity:** make the optimize route return a short keep/change/add decision before editing, and distinguish project facts from personal preferences that belong globally. This would help non-technical users avoid putting every preference into every project and would make the before-and-after value easier to demo.

### [Big Model Smell](../skills/big-model-smell/SKILL.md)

**Current strength:** a clear simplification job with sensible safety exceptions.
**Opportunity:** require a visible before-and-after operating flow plus labelled estimates for time, model cost, maintenance burden, and failure risk. Operators need to see what becomes cheaper or easier; otherwise the output can remain an abstract architecture opinion.

### [Brand System Builder](../skills/brand-system-builder/SKILL.md)

**Current strength:** broad coverage of the right visual-system decisions and accessibility.
**Opportunity:** require actual reusable outputs such as CSS variables or design tokens, type scales, colour swatches, component examples, and two applied layouts. Add a route for thin references so the model proposes directions without presenting them as established brand truth. The current written outline can be correct while still producing a generic identity.

### [Content Refresher](../skills/content-refresher/SKILL.md)

**Current strength:** preserves proof and voice while separating current facts from stale ones.
**Opportunity:** add a short content-value decision before rewriting: keep and refresh, consolidate, redirect, or retire. For the intended audience, polishing a piece that no longer has a useful reader job is wasted work. The refreshed draft should also retain one recognisable original scene, example, or opinion when supported.

### [Daydreamer](../skills/daydreamer/SKILL.md)

**Current strength:** unusually good novelty, evidence, and usefulness gates.
**Opportunity:** label the distance and confidence of each connection, then favour consequences tied to a current decision or project. This would keep the output from becoming clever-but-detached idea generation and make it more useful to an operator choosing what to do next.

### [De-Slop](../skills/de-slop/SKILL.md)

**Current strength:** concrete editing patterns and strong meaning-preservation rules.
**Opportunity:** split universal problems, such as vague claims and repetitive rhythm, from configurable voice preferences, such as em dashes, headings, or particular phrases. Several current rules reflect Nahid's voice and should not be treated as universal evidence of AI writing. Also change “top 3 recurring tells” to “up to 3 observed patterns” when only one draft is supplied.

### [Digital Lead Magnet Maker](../skills/digital-lead-magnet-maker/SKILL.md)

**Current strength:** connects the resource, opt-in copy, delivery message, and next step.
**Opportunity:** add a usefulness gate: the resource must help the reader complete a small job without buying anything else. Require one worked example, a completion state, and a finished file or production-ready handoff appropriate to the host. This prevents padded PDFs and generic checklists that do not earn trust.

### [Digital Product Launch Builder](../skills/digital-product-launch-builder/SKILL.md)

**Current strength:** good readiness checks, capacity awareness, and decision rules.
**Opportunity:** add an explicit smallest-credible-launch choice based on existing audience evidence, delivery capacity, and available proof. Label targets as user goals or test assumptions rather than forecasts. The core assets also need a small channel craft bar so they do not become one message copied across email and social.

### [Find a Power](../skills/find-a-power/SKILL.md)

**Current strength:** disciplined deterministic routing with collision handling.
**Opportunity:** when one skill clearly matches, show one concrete example of the artifact the user will receive and begin with their existing context. This makes discovery feel like progress rather than a catalogue search, which matters in a first OpenAI demo.

### [Give Feedback](../skills/give-feedback/SKILL.md)

**Current strength:** excellent payload minimisation and explicit approval boundary.
**Opportunity:** simplify the user-facing receipt and deletion-token explanation into normal language, while keeping the protocol underneath. In fallback mode, prepare the text only and clearly separate that from opening or posting a GitHub issue.

### [Instagram Carousel Maker](../skills/instagram-carousel-maker/SKILL.md)

**Current strength:** detailed export mechanics and clear capability modes.
**Opportunity:** this skill is over-specified around one fixed 420-pixel, seven-slide, light/dark template and under-specified around editorial judgment. Move code and export details into references or scripts. Put source truth, a distinct reader promise, narrative tension, proof, brand references, and layout variation first. The agent should inspect rendered slides itself when tools permit rather than asking the user to find basic visual defects. This is the most important visual-quality rewrite.

### [Landing Page CRO](../skills/landing-page-cro/SKILL.md)

**Current strength:** separates observations from hypotheses and refuses fake lift claims.
**Opportunity:** require the traffic source and visitor awareness stage when available, then trace one complete decision path from arrival to action. The skill should produce a coherent revised page hierarchy, not only isolated copy replacements, because non-technical users struggle to assemble disconnected recommendations.

### [LLM Council](../skills/llm-council/SKILL.md)

**Current strength:** honest capability tiers and genuine disagreement in the synthesis.
**Opportunity:** shorten the skill substantially and make the council adaptive. Five advisors plus five peer reviewers and two mandatory files is excessive for many decisions, and subagents using the same model are not fully independent evidence. Start with a decision packet containing facts, assumptions, options, and decision criteria; then use only the perspectives needed. Keep model provenance explicit and make the HTML report optional. Replace the hypothetical market-demand example, which models unsupported certainty.

### [Meeting Miner](../skills/meeting-miner/SKILL.md)

**Current strength:** rich modes, transcript grounding, and strong protection against invented deal outcomes.
**Opportunity:** reduce the long repeated instructions into a core evidence method plus mode references. Let the user's desired decision select depth, and return fewer insights when evidence is thin instead of filling numerical quotas. The mandatory opening sentence can also be removed so outputs sound natural in different contexts.

### [Offer Builder](../skills/offer-builder/SKILL.md)

**Current strength:** credible promises, missing-proof handling, and commitment-based validation.
**Opportunity:** include delivery economics and operator capacity: time per customer, fulfilment risk, support load, and margin assumptions. Then produce one ready-to-use test message or sales conversation outline, not only the offer card. This makes the result easier for a freelancer or small business to test immediately.

### [Operator Audit](../skills/operator-audit/SKILL.md)

**Current strength:** unusually explicit consent and privacy handling.
**Opportunity:** separate this from the public directory package or make user-supplied evidence the default product route. Local conversation-history access remains a directory-policy risk even with consent. For audience quality, add a lightweight mode based on a task export, calendar, or user-selected project notes so the useful prioritisation does not depend on broad history access.

### [Plain AI Explainer](../skills/plain-ai-explainer/SKILL.md)

**Current strength:** one of the strongest audience-specific skills in the package, with clear reading-level and factual standards.
**Opportunity:** treat the capability-risk-fix-reassurance opening as one option rather than a default formula, or repeated use will sound recognisably templated. Keep source verification prominent and make the practical next step something a non-technical operator can actually do without hidden setup.

### [Request a Power](../skills/request-a-power/SKILL.md)

**Current strength:** strong privacy minimisation and approval workflow.
**Opportunity:** make the missing-job test visible in one sentence: what the user wants, why current skills do not cover it, and the generalised request. This reassures users that the plugin is not collecting a renamed duplicate or confidential business context.

### [Start Here](../skills/start-here/SKILL.md)

**Current strength:** quickly moves from orientation to one real job.
**Opportunity:** replace the absolute statement that the user's work “never leaves the machine” with an exact distinction from the start: skill inputs and outputs stay in the agent; a small anonymous usage event may be sent unless disabled; feedback is sent only after preview and approval. Trust language should be accurate before it is reassuring.

### [Voice DNA](../skills/voice-dna/SKILL.md)

**Current strength:** evidence-based voice extraction and a portable deliverable rather than a one-off style note.
**Opportunity:** add a provisional route for fewer or shorter samples, with explicit confidence labels, and avoid claiming one skill format installs identically across every AI tool. Separate voice constants from channel-specific habits so a user's casual posts do not make their client emails artificially casual.

### [Weekly Review](../skills/weekly-review/SKILL.md)

**Current strength:** a clear recurring practice with evidence, drift, stopping, and one priority.
**Opportunity:** allow one primary outcome plus a small maintenance floor for operators who must still serve clients or fulfil commitments. Keep the forced ranking, but do not imply that all legitimate work outside the priority is failure. A carry-over line from the previous review would strengthen the compounding record.

### [What's New](../skills/whats-new/SKILL.md)

**Current strength:** carefully separates installed information from live release information.
**Opportunity:** translate each release into one or two user jobs with a concrete example and say whether an update or new session is required. Keep it short enough to act as a release tour, not a changelog reader.

### [Workflow and SOP Builder](../skills/workflow-and-sop-builder/SKILL.md)

**Current strength:** strong ownership, exception, recovery, and approval design.
**Opportunity:** show current state and proposed state separately, with labelled time or effort estimates where supplied. Add a minimum viable SOP route for solo operators so every repeated task does not become a heavyweight process. The final run checklist should remain usable without rereading the full document.

## Efficient Quality Upgrade Plan

### Before the public demo

1. Rewrite Instagram Carousel Maker around editorial and visual decisions, keeping export mechanics as reusable references.
2. Tighten LLM Council's evidence packet, capability claims, and adaptive depth.
3. Correct De-Slop's universal-versus-personal voice rules.
4. Correct Start Here's first privacy explanation.
5. Run one clean-session fixture for each changed skill using public or synthetic evidence and keep the result as a reference example.

### After the demo, before broad audience promotion

1. Upgrade Brand System Builder and Digital Lead Magnet Maker to require finished, usable artifacts.
2. Add current-state versus proposed-state handling to Workflow and SOP Builder.
3. Decide whether Operator Audit belongs in a separate private/local edition.
4. Add one realistic golden example to each high-value artifact skill and use it for regression review when the skill changes.

### Avoid

- Do not paste one universal quality checklist into all 28 skills.
- Do not require an elaborate self-audit section in every user-facing output.
- Do not solve generic work only by asking for more inputs; the skill should make the difficult decisions once it has enough evidence.
- Do not make a visually complete example the proof that a skill works. The example must also be specific, truthful, useful, and native to its intended format.

## Current Pass Decisions

- Customer Insight Synthesizer kept its existing two-item minimum, as requested.
- Content Performance now requires an explicit alternative-evidence disclosure when real metrics are absent.
- Case Study Builder now routes evidence to a full case study, project story, or evidence plan.
- Content Repurposing now includes a stronger channel craft and specificity layer.
- HTML Slideshow received light source-led visual improvements without changing its core or adding a separate output-audit framework.
