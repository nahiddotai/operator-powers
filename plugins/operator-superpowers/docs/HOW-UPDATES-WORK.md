# How Updates Work

Install once, then receive new and improved superpowers through marketplace updates.

## Where updates come from

The collection is self-improving, and here is exactly what that means: releases are decided by what users deliberately contribute. Feedback ratings, superpower requests, and anonymous usage counts all feed the roadmap: a skill rated low gets reworked, a job requested repeatedly becomes a new superpower, and usage counts show which skills earn their place. Nothing is scraped or observed silently; every signal is either a submission you approved after seeing it in full or a documented anonymous count with an off switch. The maker reviews the signals and pushes the release, and `whats-new` tells you what user input shaped it.

## How a release reaches you

What actually happens:

1. A new version is published to this repository with a release tag.
2. Your client's marketplace sees the new version when it refreshes (some clients refresh automatically, some on command).
3. You update (or auto-update applies it), then start a new session or task.
4. The one-time notice tells you the version changed; `whats-new` lists what's inside.

What does NOT happen: files do not appear instantly in your current conversation, and nothing updates silently mid-session. If an update seems missing, refresh the marketplace, update, and open a fresh session; see TROUBLESHOOTING.md.
