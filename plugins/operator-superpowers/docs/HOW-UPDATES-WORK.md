# How Updates Work

Install once, then receive new and improved superpowers through marketplace updates.

What actually happens:

1. A new version is published to this repository with a release tag.
2. Your client's marketplace sees the new version when it refreshes (some clients refresh automatically, some on command).
3. You update (or auto-update applies it), then start a new session or task.
4. The one-time notice tells you the version changed; `whats-new` lists what's inside.

What does NOT happen: files do not appear instantly in your current conversation, and nothing updates silently mid-session. If an update seems missing, refresh the marketplace, update, and open a fresh session; see TROUBLESHOOTING.md.
